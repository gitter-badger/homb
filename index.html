Homb

Welcome Homb

The Homb project is set of bash extensions for applications. It is intended for use with fresh for managing application configurations including Homb itself.

Homb only works with newer versions of bash and also relies on git. Homb can be installed solo to extend individual applications. Currently we have only tested with newer versions of bash on CentOS/Redhat and flavors of Ubuntu.

Homb Alone

For this example we install Homb into the ~/.homb directory. Here is how to clone .homb into your "${HOME}" or "~" directory.

git clone https://github.com/threeiem/homb ~/.homb

Now add this line to your ~/.bash_profile file. DO NOT ADD THIS LINE TO THE ~/.bashrc FILE! The .homb project is intended for interactive shells only.

export HOMBASE="${HOME}/.homb" && test -e $HOMBASE/rc && source $HOMBASE/rc

Homb with Fresh

The Homb project works with fresh to handle the application configuration files. Fresh is the right hand for .homb and functions poorly without fresh.

Install fresh

Install fresh with the following:

bash -c "`curl -sL get.freshshell.com`"

Configure Bash (~/.bash_profile)

Once fresh is downloaded add this to your ~/.bash_profile file (not ~/.bashrc).

# Set $HOMBASE before source of Fresh's shell.sh
export HOMBASE="${HOME}/.homb"
fpath="${HOME}/.fresh/build/shell.sh" && test -f $fpath && source $fpath

Configure (~/.freshrc)

Add this to the .freshrc file. The location of this file can be configured so we are using ~/.freshrc in the home directory.

Common Problems

  • Not using bash! Homb is for bash only. Sorry, maybe another time.

  • Non-Interactive Bash Check to see if you are running an interative shell. Check with this command...

shopt login_shell
  • Graphical Does Not Mean Interactive Some terminal programs like konsole do not start bash interactively. For konsole you will need to modify the profile. Here is the command to run bash interactively.
bash -l
